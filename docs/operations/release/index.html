    How About Automation?

    There is a partially-working ./ci/test/main.go binary that you can play with. But unless that is fixed, we‚Äôll have to run SPIKE smoke test manually.

    ./ci/test/main.go is designed for integration tests, and it assumes a working SPIKE and SPIKE environment to execute properly.

    Create a Secret

    spike secret put /acme/db user=spike pass=SPIKERocks
# Output:
# OK

    Verify secret:

    spike secret get /acme/db
# Output:
# pass: SPIKERocks
# user: spike

    Create a Policy

    spike policy create --name=workload-can-read \
  --path="/tenants/demo/db/*" \
  --spiffeid="^spiffe://spike.ist/workload/*" \
